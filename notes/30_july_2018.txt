Περιγραφή:
Το έργο θα είναι υλοποιημένο σε έναν ιστότοπο (θα είναι δλδ ένα σάιτ)
Το σάιτ θα πρέπει να έχει τα εξής components:
1. Ένα tool/data repository (κυρίως για bioinformatics / genetics). Το repository θα πρέπει να έχει τα εξής (Τα περισσότερα ΔΕΝ τα έχουν άλλα repositories)
    1a. Documentation
    1b. Ready to run (executable) bash code that installs the tool
    1c. Ready to run (executable) bash code that validates a tool installation
    1d. Μηχανισμό ο οποίος θα τρέχει τα 1b και 1c σε virtualized environments (i.e. Docker)
    1e. Dependency tree (with other tools)
    1f. Version tree
    1g. Formal (Machine Readable) Περιγραφή των παραμέτρων (input / output)
    1η. Formal (Machine Readable) Περιγραφή Quality Control (requirements, execution time, memory, disk space)
2. Workflow Management System. Το οποίο θα υποστηρίζει:
    2a. Documentation
    2b. Version Tree (multiple versions of the same Workflow)
    2c. Graphic online Workflow Editor
        2c1. Tool Dependencies should be visible on the graph
        2c2. Tool Parameters should be visible on the graph
        2c3. Workflow Building with simple Drag'n'Drop. (i.e. Drop a tool in current Workflow)
        2c4. Worklfows can call / link to other workflows (even recursively.. ). No limit on nesting.
   2d. Export / Import in various formats (i.e. WDL)
   2e. Workflows are directly executable and runnable
   2f. The system will create a report of the parameters, tools, version, execution time, results.. --> Reproducibility
3. Reputation system
   3a. Each tool / workflow is associated with the user(s) that created it
   3b. Tools workflow are rated (upvotes/downnvotes) and users can comment.
   3c. Each user has a stackexchange-like (badges) rating system. Users have a personal profile page. 
   3d. On top of this information, there will be a user/tool/workflow recommendation system that will respond to user queries (which WF is better for this task?)
   3e. Connect with ORCID
4. Citation Manager
   4a. Documentation (of Tools, Workflows, Comments) support rich test formatting that includes citations (i,.e. like this: [Kanterakis10])
   4b. The reports will include a full "References" section that will include these citations.
   4c. Citations are indexed. Users can search on citations as well as other components (Workflows, Tools)
5. REST APIs
   5a.  Θέλουμε το "σύστημα" να μπορεί να "παίζει" και από άλλα περιβάλλοντα (π.χ. Galaxy) χωρίς να χρειάζεται web interaction.

Ίσως να ξεχνάω κάποια features αλλά αυτά είναι αρχικά για να σας δώσω μια ιδέα για το.. όραμα του OpenBioC.
